Linux 进程

2017年4月27日

脱离终端

进程的简单理解

进程 ：正在运行 执行的程序
并发 ：同时运行很多个进程
比如一个用户访问了我们的网站，那么至少会产生一个进程
比如Apache，好像是最多产生2000个Apache进程
所以并发量太大，超过了最大的进程量，服务器就可能死机
服务器的瓶颈 在于内存和CPU
内存不够 加点内存。
CPU不够多买几台服务器 做个集群（多台服务器）。
理解Linux下的进程管理，可以类比window下的任务管理器，里面也有进程管理， 结束进程（相信很多人都用过这个功能吧）

ls 也会产生进程
只是执行速度太快，且执行完就马上消失了

查看进程命令：

ps aux 
ps -le

两者都可查看进程，我一般使用ps aux
进程 常驻在内存中

top
父进程

没有正确退出
内存 硬盘 CPU 缓冲
top -b -n 1 >top.log

杀死进程

单一
kill -l
kill -HUP 1222 平滑

kill all
kill -1 2111 重启进程
kill -9 强制杀死进程
kill all -i -9 httpd -i 交互式 问需不需要 -I
